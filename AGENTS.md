This project is an attempt at a modern take on what agentic web browsing and interaction can become. The goal is to have a CEF visual panel library that is then driven by agents using React to create dynamic visual interfaces. This library will itself take data and call functions from a guided API surface using similar underlying concepts to MCP but without the bloat of full MCP. The API surface will be called a Semantic Store and contains both resources and functions in a guided experience similar to how the best MCP servers work so that agents are directed how to use the website through the Semantic Store. The visual panel windows driven by React will be called Visual Panel's and there will also be Compute Modules which are very short Rust scripts that interact with the Store to do things so that agents can later call them. Agents should primarily use the Store itself directly instead of simply making Modules which are more meant for solving repetitive tasks in a deterministic manner. Lastly there is the Data Lens which is an optional filter that agents can create to sit between a Store and a View so that highly personalised functionality can be created for a Store without having to modify it or run code on the React layer.

The tools to use are Rust for Semantic Stores, Rust for CEF + native Window agentic bindings, React for the actual content of a Visual Panel and data fetching, Rust for the Data Lens and Rust for Compute Module's. BAML owns the agentic logic and the complete routed voice-turn flow, including its decisions, branches, retries, and task reselection. The Python manager, called the Agent Host, owns process setup, audio processing, data retrieval, persistence, TTS playback, and other external I/O. Python hands each routed voice turn to one top-level BAML flow and exposes typed boundary calls; it must not recreate the agentic control flow in Python.

There will be a graph db for memory storage as well. There will be nodes for various memories including descriptions, tagging and creation/read/modify timestamps that and a tree structure allowing agents to search for the most recent memories relevant to the task at hand and keep those in the context for performing the task. This should allow for the agentic flow to memorise when you ask for things like always displaying a View in a certain way for a website or a global memory for how fast you like things to scroll, etc...

This will use the sounddevice Python library for taking audio in. Python library pocketsphinx for local trigger word detection such as "jarvis do x" and phrase detection such as "conversation on". Python library silero-vad for voice activity detection. Python library faster-whisper and whisper.cpp for transcribing audio prompts directly to BAML agentic flow. Python library kokoro-onnx for TTS voice playback using the af_heart voice. sounddevice should have the ability to cancel audio playback when the user starts talking but not completely destroy the underlying tasks it was completing.

Don't make large files. Files should be named by intent and be small enough to view on a 1080p monitor without lots of scrolling.
